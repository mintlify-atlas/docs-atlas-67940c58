---
title: "Categorias API"
description: "API endpoints for managing categories (categorías) in the Cinefinder application"
---

The Categorias API allows you to create, read, update, and delete category records in the Cinefinder system.

## Get All Categories

<CodeGroup>
```bash cURL
curl -X GET http://localhost:8080/categorias
```
</CodeGroup>

Retrieves a list of all categories in the system.

### Response

<ResponseField name="[]" type="array">
  Array of category objects
  
  <Expandable title="Category Object">
    <ResponseField name="id" type="integer">
      Unique identifier for the category
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Name of the category
    </ResponseField>
  </Expandable>
</ResponseField>

### Response Example

```json
[
  {
    "id": 1,
    "nombre": "Drama"
  },
  {
    "id": 2,
    "nombre": "Comedy"
  }
]
```

## Get Category by Name

<CodeGroup>
```bash cURL
curl -X GET http://localhost:8080/categorias/{nombre}
```
</CodeGroup>

Retrieves a specific category by its name.

### Path Parameters

<ParamField path="nombre" type="string" required>
  The name of the category to retrieve
</ParamField>

### Response

<ResponseField name="id" type="integer">
  Unique identifier for the category
</ResponseField>

<ResponseField name="nombre" type="string">
  Name of the category
</ResponseField>

### Response Example

```json
{
  "id": 1,
  "nombre": "Drama"
}
```

---

## Create Category

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8080/categorias \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Thriller"
  }'
```
</CodeGroup>

Creates a new category in the system.

### Request Body

<ParamField body="nombre" type="string" required>
  Name of the category
</ParamField>

### Response

Returns the created category object with HTTP status 201 (Created).

<ResponseField name="id" type="integer">
  Unique identifier for the newly created category
</ResponseField>

<ResponseField name="nombre" type="string">
  Name of the category
</ResponseField>

### Response Example

```json
{
  "id": 3,
  "nombre": "Thriller"
}
```

---

## Update Category

<CodeGroup>
```bash cURL
curl -X PUT http://localhost:8080/categorias/{id} \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Suspense"
  }'
```
</CodeGroup>

Updates an existing category.

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the category to update
</ParamField>

### Request Body

<ParamField body="nombre" type="string" required>
  Name of the category
</ParamField>

### Response

Returns the updated category object.

<ResponseField name="id" type="integer">
  Unique identifier for the category
</ResponseField>

<ResponseField name="nombre" type="string">
  Name of the category
</ResponseField>

### Response Example

```json
{
  "id": 3,
  "nombre": "Suspense"
}
```

---

## Delete Category

<CodeGroup>
```bash cURL
curl -X DELETE http://localhost:8080/categorias/{id}
```
</CodeGroup>

Deletes a category from the system.

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the category to delete
</ParamField>

### Response

<ResponseField name="type" type="string">
  Response type, always "Success"
</ResponseField>

<ResponseField name="code" type="string">
  HTTP status code (200)
</ResponseField>

<ResponseField name="details" type="string">
  Success message with the deleted category ID
</ResponseField>

<ResponseField name="location" type="string">
  Request URI that was called
</ResponseField>

<ResponseField name="timestamp" type="string">
  Timestamp of the operation
</ResponseField>

### Response Example

```json
{
  "type": "Success",
  "code": "200",
  "details": "la categoría con el id: 3. fue eliminada correctamente.",
  "location": "/categorias/3",
  "timestamp": "2026-03-04T10:30:00"
}
```
